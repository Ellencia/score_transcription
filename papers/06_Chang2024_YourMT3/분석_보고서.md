# YourMT3+: Multi-instrument Music Transcription with Enhanced Transformer Architectures and Cross-dataset Stem Augmentation 분석 보고서

## 핵심 요약

YourMT3+는 MT3의 **언어 토큰 디코딩(language token decoding)** 방식을 계승하면서 인코더 구조·학습 기법·데이터 활용을 전면 강화한 다중악기 채보 모델군이다. 다성(polyphonic) 음악을 악기별 악보로 변환하는 과제는 여러 악기를 동시에 식별하고 음높이·정밀 타이밍을 받아 적어야 하는 데다, 완전 주석(fully annotated) 데이터가 부족해 학습이 어렵다. 저자들은 이 두 축 모두를 공략한다.

구조 측면에서는 (1) 시간-주파수 영역의 **계층적 어텐션 Transformer(PerceiverTF)**로 인코더를 교체하고, (2) **전문가 혼합(Mixture of Experts, MoE)**을 도입해 다중 데이터셋 학습에서 과제별 인코딩을 가능하게 했다. 데이터 측면에서는 (3) 불완전 주석으로도 학습 가능한 **다채널 디코딩(multi-channel decoding)**과, (4) 데이터셋을 섞는 **인트라/크로스 스템 증강(intra-/cross-stem augmentation)**을 제안했다.

가장 주목할 결과는 **보컬을 음원 분리(voice separation) 전처리 없이 직접 채보**한다는 점이다. 분리 없이 노래를 채보할 때 기본 YMT3는 F1 3.6%에 그쳤으나, 크로스 스템 증강을 적용한 모델은 **64%**까지 도달했다. 10개 공개 데이터셋 벤치마크에서 최종 모델(YPTF.MoE+Multi)은 MT3와 PerceiverTF를 다중악기 데이터셋(URMP·Slakh)에서 크게 앞섰다. 다만 상용 팝 음악 녹음 테스트에서는 현재 모델들의 한계가 드러났다.

## 상세 요약

YourMT3+는 모델 계보로 이해하면 명확하다. **YMT3**는 MT3를 처음부터 재현한 베이스라인이고, **YPTF+Single**은 인코더를 PerceiverTF로 바꾸고 스템 증강(+)을 더한 것, **YPTF.MoE**는 인코더 FFN을 MoE로 교체한 것, 최종 **YPTF.MoE+Multi**는 이 모든 요소에 다채널(Multi) 디코더를 통합한 것이다. 전체 파라미터 증가는 MT3 대비 2.5% 미만으로 작다.

**인코더(PerceiverTF)**: 각 블록은 지역(local)·시간(temporal) 트랜스포머 서브블록으로 구성되며, 학습 가능한 잠재 배열(latent array)에 대한 **스펙트럼 교차 어텐션(spectral cross attention)**을 사용한다. 12개 악기 + 가창을 위해 잠재 수 k=26(악기 그룹 수의 2배)으로 두고, 각 잠재 쌍이 해당 악기 그룹의 쿼리 역할을 한다. 위치 인코딩으로 RoPE, 정규화로 pre-RMSNorm을 채택했다.

**MoE**: 잠재·시간 트랜스포머 블록의 FFN을 8개 전문가 중 2개로 라우팅하는 Mixtral 방식 MoE로 교체했다. 전문가 2개가 1개·4개보다 좋았으며, 복잡도는 약 5% 증가에 그쳤지만 모든 데이터셋에서 일관된 성능 향상을 보였다.

**다채널 디코딩(Multi decoder)**: MT3의 Single 디코더는 여러 악기를 한 시퀀스에 뒤섞어 출력하므로, 일부 악기만 주석된 데이터에서 미주석 악기의 손실을 마스킹하기 어렵다. Multi 디코더는 PerceiverTF의 잠재를 채널(k′=13)로 나눠 **악기 그룹별로 독립 디코딩**(파라미터 공유 병렬 디코더)함으로써, 부분 주석 데이터를 활용하고 과제-쿼리 기반 학습을 가능하게 한다.

**스템 증강**: 인트라 스템 증강은 한 세그먼트 내 악기를 Bernoulli 확률(기본 p=0.7)로 무작위 드롭한다. 크로스(데이터셋 간) 스템 증강은 서로 다른 데이터셋의 스템을 섞어 새로운 혼합을 만든다(지수 감쇠 생존함수로 최대 5개 세그먼트까지 결합). 이 증강이 **분리 전처리 없는 보컬 채보의 핵심**이다.

## 방법론과 데이터

입력은 6초를 쓰는 PerceiverTF보다 짧은 **2.048초 세그먼트**(16kHz)다. YMT3는 256×512 로그 mel 스펙트로그램을, YPTF는 110×1024 로그 스펙트로그램을 2D ResNet 프리인코더로 처리해 110×128×128 형태로 만든다. 출력은 MIDI-유사 토큰이며, 프로그램 100·101을 각각 **가창(멜로디)·가창(코러스)**으로 예약했다. 학습은 NVIDIA A100 2장, BFloat16, AdamWScale + 코사인 스케줄러로 300K step.

| 항목 | 내용 |
|---|---|
| 베이스라인 인코더 | T5-small v1.1 (8블록, 6-head) — YMT3 |
| 제안 인코더 | PerceiverTF (local+temporal, SCA, 잠재 k=26, RoPE) |
| MoE | FFN→Mixtral MoE (8전문가 중 2개 라우팅) |
| 디코더 | T5 기반 Single(N=1024) 또는 Multi(채널별 N=256, 병렬) |
| 입력 | 2.048초 / 16kHz / YPTF: 110×1024 스펙트로그램 + ResNet |
| 증강 | 인트라 스템(p=0.7) + 크로스 데이터셋 스템 + GPU 피치시프트(±2반음) |
| 학습 | A100×2, BFloat16, AdamWScale, 300K step, dropout 0.05 |
| 데이터셋 | 10개 공개 (학습/테스트 일부 상이) |

학습 데이터셋: MusicNet-EM, GuitarSet, MIR-ST500, ENST-Drums, Slakh, EGMD, Maestro, CMedia, URMP, SMT-Bass. 테스트: MusicNet, GuitarSet, MIR-ST500, ENST-Drums, Slakh, Maestro, MAPS, URMP, RWC-Pop(주석 정제). RWC-Pop에 대한 다중악기 AMT 성능을 처음으로 조사했다.

```mermaid
flowchart LR
    A["다중 데이터셋 스템\n(보컬 포함)"] --> B["인트라/크로스 스템 증강\n+ 피치시프트"]
    B --> C["스펙트로그램 + 2D ResNet"]
    C --> D["PerceiverTF 인코더\n(SCA · MoE · RoPE)"]
    D --> E["Multi 디코더\n(악기 그룹별 병렬, 마스킹 가능)"]
    E --> F["악기별 MIDI\n+ 보컬 직접 채보"]
```

## 결과와 의의

10개 데이터셋의 Note Onset F1에서 최종 모델(YPTF.MoE+Multi)은 MT3·PerceiverTF 베이스라인을 다중악기에서 크게 앞섰다(아래 발췌, 출처: 논문 Table 2·Table 3).

| 데이터셋 / 지표 | YMT3(=MT3 재현) | MT3(colab) | PerceiverTF | YPTF.MoE+Multi |
|---|---|---|---|---|
| Slakh — Onset F1 (Agnostic) | 64.8 | 75.20 | 81.9 | **84.56** |
| Slakh — Multi F1 (All) | 57.69* | 57.69 | 62.0 | **74.84** |
| URMP — Onset F1 (Agnostic) | – | 76.65 | 77.0 | **81.79** |
| URMP — Multi F1 (Ensemble) | – | 58.71 | 59.0 | **67.98** |
| GuitarSet — Guitar Onset F1 | – | – | 91.10 | **91.65** |
| ENST-Drums (DTM) — Drum F1 | – | – | 79.00 | **87.27** |
| MIR-ST500 — Singing(50ms) | 3.62 | – | – | **71.07** |
| MIR-ST500 — Singing(100ms) | – | – | 78.50 | **82.08** |

(*Slakh Multi F1의 MT3 값. Agnostic=악기 무관, Multi=악기 일치까지 요구하는 더 엄격한 지표)

Slakh 구성 분석(Table 3)에서 YMT3 베이스(Onset 64.8 / Offset 41.7 / Drum 77.8)에 인트라 증강·전체어휘·데이터균형·크로스 증강·PTF 인코더·MoE·Multi 디코더를 차례로 더해 최종 **84.6 / 70.7 / 90.1**에 도달했다. MoE는 모든 데이터셋에서 일관된 향상을 보였다.

가장 큰 의의는 **보컬 직접 채보**다. 음원 분리(SVS) 전처리 없이 노래를 채보할 때 YMT3는 F1 3.6%에 불과했으나, 크로스 스템 증강 적용 시 64%로 도약했다. 이는 "보컬 분리기 → 채보기"의 2단계 파이프라인을 단일 모델로 대체할 수 있음을 시사한다. 또한 추론 속도도 실용적이어서, Ampere 이상 GPU에서 float16 기준 약 36배 실시간(약 6분 피아노를 10초 내 처리)으로 동작한다.

## 한계와 비판

- **상용 팝 음악에서의 붕괴**: RWC-Pop 테스트에서 피아노·베이스·보컬·드럼을 제외한 비주류 악기는 F1 10% 미만으로 떨어졌다. 저자들은 합성(synthetic) 데이터 위주 학습이 팝의 다양한 음색을 못 담은 편향 탓으로 본다.
- **보컬 타이밍 정밀도**: MIR-ST500에서 50ms 기준 성능이 베이스라인(PerceiverTF)에 다소 못 미친다. 많은 onset 오차가 50ms를 넘어 100ms 범위에 들어가, 100ms 기준에선 약 10% 더 높지만 50ms 정밀도는 뒤처진다.
- **옥타브 오류**: 피아노를 제외한 음정 악기에서 chroma 단위 지표와의 격차가 커, 상당한 옥타브 오류가 존재한다. 더 다양한 피치시프트가 도움될 수 있다고 인정했다.
- **합성 데이터 의존**: 다양한 악기 학습이 Slakh 같은 합성 데이터에 의존해, 실제 녹음의 음색 분포와 괴리가 있다.
- **피치시프트 부작용**: PS로 학습한 모델이 일부 구간을 반음 위/아래로 잘못 채보하는 알려진 추론 이슈가 있다(noPS 모델에선 미발생).
- **MoE·다채널의 복잡도**: 구조가 MT3보다 복잡해 재현·튜닝 부담이 있으나, 저자들이 완전 재현 코드를 공개해 일부 완화했다.
