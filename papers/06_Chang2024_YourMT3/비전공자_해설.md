# YourMT3+: Multi-instrument Music Transcription — 비전공자 해설

## 이 논문이 풀려는 문제는 무엇인가

MT3는 여러 악기를 한 모델로 받아 적는 길을 열었지만, 두 가지 큰 숙제를 남겼습니다. 첫째, **노래(보컬)를 직접 받아 적지 못했습니다.** 가사 멜로디를 채보하려면 보통 "반주에서 목소리만 떼어내는 분리기"를 먼저 돌린 뒤 채보기에 넣어야 했죠 — 두 단계라 번거롭고 오류도 쌓입니다. 둘째, **악기마다 정답 데이터의 양이 천차만별**이라, 일부 악기만 라벨이 붙은 "반쪽짜리 정답" 데이터를 제대로 못 썼습니다.

YourMT3+는 MT3를 토대로 이 숙제들을 정면으로 풉니다. 더 똑똑한 귀(인코더)를 달고, 반쪽짜리 데이터도 학습에 쓰는 요령을 고안하고, 무엇보다 **보컬을 분리기 없이 곧바로 받아 적게** 만들었습니다.

## 한 줄 비유로 본 핵심

**"여러 전문 통역사를 한 팀으로 두고, 부족한 녹음은 짜깁기로 늘린다."** YourMT3+는 악기별 전문가(Mixture of Experts)를 두어 각 악기를 더 잘 알아듣고, 서로 다른 곡의 악기 트랙을 레고처럼 섞어(스템 증강) 부족한 학습 데이터를 무한정 만들어냅니다. 그 덕분에 노래까지 분리기 없이 바로 받아 적습니다.

## 핵심 아이디어를 한 그림으로

```mermaid
flowchart LR
    A["여러 곡의 악기·보컬 트랙"] --> B["레고처럼 섞기\n(스템 증강)"]
    B --> C["똑똑한 귀\n(PerceiverTF + 전문가팀 MoE)"]
    C --> D["악기 그룹별 병렬 받아쓰기\n(반쪽 데이터도 OK)"]
    D --> E["🎸🥁🎤 악기별 + 보컬\n악보(MIDI)"]
```

## 알아야 할 핵심 용어

| 용어 | 영문 | 직관적 설명 |
|---|---|---|
| 다중악기 채보 | Multi-instrument transcription | 여러 악기 음악을 악기별로 받아 적기 |
| 스템 | Stem | 곡을 이루는 개별 악기(트랙). 보컬 스템, 드럼 스템 등 |
| 스템 증강 | Stem augmentation | 여러 곡의 스템을 섞어 새 학습곡을 만드는 데이터 늘리기 |
| 전문가 혼합 | Mixture of Experts (MoE) | 여러 "전문가" 망 중 상황에 맞는 둘만 골라 쓰는 기법 |
| PerceiverTF | PerceiverTF | 시간·주파수를 함께 보는 똑똑한 인코더(귀) |
| 다채널 디코딩 | Multi-channel decoding | 악기 그룹마다 따로 받아 적어 반쪽 데이터도 쓰는 방식 |
| 음원 분리 | Voice/Source separation | 반주에서 목소리만 떼어내는 전처리 (여기선 불필요!) |
| 보컬 채보 | Vocal transcription | 노래 멜로디를 음표로 받아 적기 |
| F1 점수 | F1 score | 정확도 채점표(0~100%) |

## 어떻게 작동하는가

1. **더 똑똑한 귀 (PerceiverTF 인코더)**: MT3의 평범한 인코더를 시간과 주파수를 동시에 살피는 PerceiverTF로 교체했습니다. 악기 그룹마다 전용 "주의 슬롯"(잠재)을 두어, 섞인 소리에서 악기를 더 잘 가려냅니다.

2. **전문가 팀 (MoE)**: 신경망 일부를 "8명의 전문가 중 상황에 맞는 2명만 호출"하는 구조로 바꿨습니다. 드럼이 나오면 드럼 전문가가, 현악이 나오면 현악 전문가가 활약하는 식이죠. 계산량은 5%만 늘었는데 모든 데이터셋에서 성적이 올랐습니다.

3. **레고 짜깁기 (스템 증강)**: 학습 데이터가 부족하니, 서로 다른 곡의 악기 트랙을 무작위로 섞어 **새로운 가상의 합주곡**을 끝없이 만들어냅니다. A곡의 드럼 + B곡의 기타 + C곡의 보컬을 합치는 식입니다. 이게 보컬 직접 채보의 비결입니다 — 다양한 조합으로 "목소리 + 다른 악기" 상황을 잔뜩 학습시키니, 굳이 분리기를 거치지 않아도 노래를 받아 적게 됩니다.

4. **반쪽 데이터 활용 (다채널 디코딩)**: 어떤 데이터는 드럼 정답만, 어떤 데이터는 보컬 정답만 있습니다. MT3는 모든 악기를 한 줄에 뒤섞어 출력해서 이런 반쪽 데이터를 다루기 어려웠죠. YourMT3+는 **악기 그룹마다 출력 줄을 따로** 두어, 정답이 있는 부분만 골라 학습합니다.

5. **노래도 한 식구로**: 출력 토큰에 "가창(멜로디)"과 "가창(코러스)" 전용 꼬리표를 추가해, 보컬을 다른 악기와 똑같이 한 모델 안에서 처리합니다.

## 왜 중요한가

가장 극적인 성과는 **보컬 직접 채보**입니다. 분리기 없이 노래를 받아 적을 때, 기존 방식(YMT3)은 F1 3.6%로 거의 못 했지만, 스템 증강을 적용하자 **64%**까지 치솟았습니다. "분리기 → 채보기"의 2단계 파이프라인을 단일 모델로 줄일 수 있게 된 것이죠. 100ms 기준 보컬 채보는 82.08%로 기존 최고(78.5%)도 넘어섰습니다.

다중악기 성적도 전반적으로 MT3와 PerceiverTF를 앞섭니다. 예컨대 다악기 합성곡 Slakh에서 "악기까지 맞히는" 엄격한 Multi F1이 MT3의 57.69%에서 **74.84%**로, 합주(URMP)는 58.71%에서 **67.98%**로 올랐습니다. 게다가 코드와 데이터를 전부 공개해 누구나 재현할 수 있고, 최신 GPU에서 실시간의 약 36배 속도로 돌아갈 만큼 실용적입니다.

다만 저자들은 정직하게 **한계도 드러냅니다.** 실제 상용 팝 음악에 넣어보니, 피아노·베이스·보컬·드럼을 뺀 나머지 악기는 정확도가 10% 아래로 무너졌습니다. 학습에 쓴 데이터가 주로 합성음이라, 진짜 팝의 다채로운 음색을 따라가지 못한 탓입니다. 이는 곧 **"다음 과제는 진짜 음악 데이터"**임을 보여줍니다.

요약하면 YourMT3+는 Hawthorne(2021) → MT3(2022)로 이어진 "채보를 번역처럼 푸는" 계보의 가장 진화한 형태입니다. 여러 악기를 넘어 **사람 목소리까지 한 모델로** 받아 적게 만든, 다중악기 채보의 현재 최전선입니다.
