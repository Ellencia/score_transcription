# Sequence-to-Sequence Piano Transcription with Transformers 분석 보고서

## 핵심 요약

이 논문은 자동 음악 채보(Automatic Music Transcription, AMT)를 **기계번역(machine translation)처럼 재정의**한 분기점에 해당한다. 그 이전까지 피아노 채보의 최고 성능 모델들(예: Onsets and Frames)은 음의 시작(onset)·지속(frame)·세기(velocity)를 각각 따로 예측하는 전용 신경망 스택과 복잡한 후처리 디코딩을 손수 설계해야 했다. 저자들은 이런 도메인 특화 설계를 모두 걷어내고, NLP에서 검증된 **범용 인코더-디코더 Transformer**(T5 구성을 거의 그대로 사용)에 스펙트로그램을 입력하면 곧바로 MIDI-유사 이벤트 토큰 시퀀스를 출력하도록 학습시켰다.

핵심 결과는 "특별할 것 없는(off-the-shelf) 일반 모델 + 단순 greedy 디코딩"만으로도 전용 설계 모델과 **동등하거나 더 나은 성능**을 낸다는 것이다. MAESTRO V1.0.0에서 Onset F1 95.95%, Onset+Offset+Velocity F1 82.18%를 기록하여 당대 최첨단(Kong et al. 2020의 80.92%)을 Onset+Offset+Velocity 기준으로 능가했다. 파라미터는 54M으로 Onsets and Frames(28M)의 약 2배에 불과하다.

이 단순성의 의의는 단지 피아노에 그치지 않는다. 입출력 토큰 정의만 바꾸면 같은 모델을 "onset만 채보" 같은 다른 과제로 즉시 전환할 수 있다는 점에서, 저자들은 향후 MIR(Music Information Retrieval) 연구의 무게중심이 "맞춤형 모델 설계"에서 "데이터셋 구축과 라벨링"으로 옮겨갈 수 있음을 시사했다. 실제로 이 접근은 이듬해 다중악기 모델 MT3로 직접 확장되었다.

## 상세 요약

전통적 피아노 채보는 오디오를 **피아노 롤(piano roll) 행렬**(시간×음높이 격자)로 변환하는 프레임 단위 분류 문제로 다뤄졌다. Onsets and Frames 류 모델은 onset 검출기·frame 검출기·velocity 검출기를 분리하고, "onset 확률이 0.5를 넘지 않으면 음을 시작하지 않는다" 같은 하드 코딩된 디코딩 규칙으로 출력을 결합했다. 이런 구성은 성능은 좋지만 음악 도메인에 종속적이고, 새 특징(예: offset)을 추가하려면 출력 헤드·손실함수·디코딩 알고리즘을 모두 새로 설계해야 했다.

이 논문의 발상은 채보를 **"오디오 → 기호 토큰" 번역**으로 보는 것이다. 입력은 스펙트로그램 프레임 시퀀스, 출력은 가변 길이의 MIDI-유사 이벤트 토큰 시퀀스다. 인코더는 self-attention으로 입력 전체를 처리하고, 디코더는 causal self-attention과 인코더에 대한 cross-attention으로 토큰을 autoregressive하게 한 개씩 생성한다. 출력 길이가 입력 오디오를 기술하는 데 필요한 토큰 수에 따라 자동으로 결정된다는 점이 핵심이다.

특히 **시간을 절대(absolute) 표현**으로 인코딩한 점이 결정적이었다. 이전의 이벤트 기반 표현은 "직전 이벤트로부터 경과한 상대 시간(relative time shift)"을 썼는데, 시퀀스 초반의 작은 오차가 누적되어 전체 정렬이 어긋난다. 절대 시간(세그먼트 시작점부터 10ms 단위로 양자화)을 쓰면 각 타임스탬프를 독립적으로 맞히면 되므로 모델 부담이 줄고 성능이 크게 향상된다(상대 시간 사용 시 Onset F1이 80.02%로 급락).

## 방법론과 데이터

모델은 T5 "small" 구성(T5.1.1 레시피)을 거의 그대로 채택했다. 연속값인 스펙트로그램을 입력하기 위해 각 프레임을 임베딩 공간으로 사영하는 dense layer를 추가했고, T5의 로그 스케일 상대 위치 버킷 대신 **고정 절대 위치 임베딩**을 사용했다. 학습은 Adafactor optimizer, batch size 256, learning rate 1e-3로 32 TPUv3 코어에서 400K step(약 2.5일) 진행했다. 데이터셋은 약 200시간의 비르투오소 피아노 연주가 정밀 정렬된 **MAESTRO**다(이전 연구 비교는 V1.0.0, 자체 ablation은 V3.0.0).

| 항목 | 내용 |
|---|---|
| 모델 | 인코더-디코더 Transformer (T5 small, d_model=512, d_ff=1024, 6-head, 인코더·디코더 각 8층) |
| 파라미터 | 54M (Onsets and Frames 28M의 약 2배) |
| 입력 | 16kHz 오디오, FFT 2048, hop 128, 512 mel-bin 로그 스펙트로그램 (입력 512 포지션) |
| 출력 토큰 | Note(128), Velocity(128, 0=note-off), Time(6000, 10ms 양자화), EOS — 출력 최대 1024 포지션 |
| 세그먼트 | 최대 약 4.088초 단위로 분할, 비중첩 디코딩 후 이어붙임 |
| 디코딩 | greedy autoregressive (각 step 최대 확률 토큰 선택, EOS까지) |
| 데이터 | MAESTRO V1.0.0/V3.0.0 (약 200시간 피아노) |

    A["피아노 오디오"] --> B["로그 mel 스펙트로그램\n(프레임 시퀀스)"]
    B --> C["Transformer 인코더\n(self-attention)"]
    C --> D["Transformer 디코더\n(cross-attention, greedy)"]
    D --> E["MIDI-유사 토큰\nNote / Velocity / Time / EOS"]
    E --> F["세그먼트 이어붙이기\n→ 음표 시퀀스(MIDI)"]

## 결과와 의의

MAESTRO V1.0.0 테스트셋에서, 제안 모델은 도메인 특화 설계 없이도 최첨단과 경쟁하는 F1을 달성했다. 특히 음악적으로 가장 까다로운 Onset+Offset+Velocity 기준에서 기존 최고치를 넘어섰다.

| 모델 | Onset, Offset & Velocity F1 | Onset & Offset F1 | Onset F1 |
|---|---|---|---|
| **Transformer (본 논문)** | **82.18** | 83.46 | 95.95 |
| Kong et al. 2020 | 80.92 | 82.47 | **96.72** |
| Kwon et al. 2020 | – | 79.36 | 94.67 |
| Kim & Bello 2019 | 80.20 | 81.30 | 95.60 |
| Hawthorne et al. 2019 (Onsets & Frames) | 77.54 | 80.50 | 95.32 |

(수치 출처: 논문 Table 1, MAESTRO V1.0.0 테스트셋)

Ablation(V3.0.0)에서는 (a) 라벨 토큰만 바꿔 "onset 전용" 과제로 전환해도 Onset F1 96.13%로 잘 작동했고, (b) 로그 mel 입력이 STFT·원시 샘플 입력보다 우수했으며, (c) 절대 시간이 상대 시간보다 압도적으로 좋았고(82.75 vs 66.25 Onset+Offset+Velocity F1), (d) 모델을 "base"(213M)로 키우자 곧바로 과적합되어 오히려 성능이 떨어졌다. 즉 피아노 채보는 복잡하지만 거대한 Transformer가 필요하지 않다는 점을 보였다.

의의는 명확하다. **AMT를 seq2seq 번역 문제로 재정의**함으로써, 전용 아키텍처 없이도 SOTA에 도달할 수 있음을 입증했고, 입출력 토큰 설계만으로 과제를 유연하게 바꿀 수 있게 했다. 이는 곧 다중악기·다중데이터셋으로 확장하는 MT3의 토대가 되었다.

## 한계와 비판

- **세그먼트 분할의 인위성**: O(n²) 어텐션 메모리 때문에 오디오를 약 4초 단위로 잘라 독립 디코딩한다. 세그먼트 경계를 가로지르는 음(note)의 처리가 취약하며, 저자들도 sparse attention으로 전곡을 한 번에 처리하는 것을 향후 과제로 남겼다.
- **피아노 단일 악기에 한정**: 본 논문은 피아노만 다룬다. 다중악기 일반화는 후속 연구(MT3)로 미뤄졌다.
- **실시간성 부재**: autoregressive 디코딩과 비교적 큰 모델로 인해 모바일·웹 실시간 구동은 distillation 등 추가 연구가 필요하다고 명시했다.
- **사전학습 미활용**: 대규모 비라벨 데이터 사전학습 없이 라벨 데이터만으로 처음부터 학습했다. 저자들은 사전학습이 도움이 될 가능성을 열어두었으나 검증하지 않았다.
- **시간 분해능 10ms**: 인간 지각 한계를 근거로 했으나, 더 미세한(예: 5ms) 또는 연속 시간 예측이 성능을 더 높일 여지가 있다고 인정했다.
